01 — OVERVIEW

Product Overview

Raj Electronics is a home appliances dealer based in Hyderabad dealing in brands such as Samsung, LG, Whirlpool, Haier, Voltas, and others. The business receives 2,000+ units per month across multiple godowns and dispatches them to retail customers and dealers.

Currently there is no system to track individual serial numbers from receipt to sale. This leads to inventory discrepancies, difficulty tracing sold units, and no audit trail for staff accountability.

This product is a Progressive Web App (PWA) + native Android app that gives Raj Electronics complete serial-number-level tracking from the moment a unit enters a godown to the moment it leaves, with a print-ready GST dispatch challan generated at the point of outward movement.

💡
Single source of truth

Every scan — inward or outward — writes in real time to a structured Google Sheet that acts as the live database. The admin can open the Sheet at any time and see the exact status of every serial number in the business.

02 — GOALS & OBJECTIVES

Goals & Objectives

Primary Goals

  • 1
    Track every unit from inward receipt to outward dispatch using its unique serial number — no unit should be unaccounted for at any point.
  • 2
    Enable staff to scan serial numbers using the phone camera (barcode/QR) without manual typing errors, at the speed required for 2,000+ units/month.
  • 3
    Auto-generate a print-ready GST dispatch challan at the time of every outward movement containing all serial numbers, dealer details, and tax breakdown.
  • 4
    Maintain a live, searchable Google Sheet that the admin can access from any device at any time — no separate software, no local database dependency.
  • 5
    Enforce a strict role-based access system where staff can only perform inward and outward operations, and all other data remains admin-only.

Success Metrics

  • 100% of inward stock has a serial number entry before being placed in godown
  • Zero untracked outward movements — every dispatch has a challan
  • Admin can locate any serial number's current status in under 10 seconds
  • Staff can scan and log a batch of 20 units in under 3 minutes
  • System handles 2,000+ serial entries per month without performance degradation
03 — USER ROLES

User Roles & Access

The system has exactly two roles. There is no self-registration — all accounts are created by the admin.

ADMIN

Business Owner / Manager

Single admin account with a fixed username and password set at system setup. The admin has unrestricted access to all features, data, reports, and settings.

  • Creates and manages employee accounts
  • Views full inventory and logs
  • Exports reports and CSV data
  • Configures godowns, GST, company info
  • Views activity log of all staff actions
EMPLOYEE

Warehouse / Dispatch Staff

Employee IDs are generated by the admin. Each employee logs in with their ID and a password set by the admin. Employees see only the Inward and Outward tabs — nothing else.

  • Can submit inward stock entries
  • Can submit outward dispatch entries
  • Can print dispatch challan
  • Cannot view inventory list or reports
  • Cannot change settings or other data

04 — AUTHENTICATION

Authentication System

Admin Login

  • Single admin account — username and password configured at first-time setup
  • Login screen is the first screen of the app — no bypass allowed
  • Admin credentials stored securely (hashed password) — not in Google Sheets
  • Session persists for 8 hours then requires re-login
  • Admin can change their own password from Settings

Employee Account Management

  • Admin creates employee accounts from the Admin Panel → Staff Management tab
  • System auto-generates a unique Employee ID in format: RE-YYYY-XXXX (e.g. RE-2024-0047)
  • Admin sets a password for each employee at creation time
  • Admin can deactivate or delete any employee account instantly
  • Deactivated employees are immediately logged out and blocked from re-entry
  • Employee passwords can be reset by admin at any time
  • Employees cannot change their own password — admin controls this

Login Screen Requirements

  • Single unified login screen for both admin and employees
  • Fields: Employee/Admin ID, Password
  • On login, role is detected automatically — admin sees full app, employee sees restricted view
  • Failed login after 5 attempts locks the account for 15 minutes
  • No "forgot password" for employees — admin resets it
⚠️
No self-registration

There must be no "Sign up" or "Create account" option visible anywhere in the app. All accounts are admin-created only. The login screen should only show ID and password fields.

05 — INWARD STOCK

Inward Stock Entry

Used when stock arrives from a supplier and is being received into a godown. This creates a new record for every serial number in the system with status IN STOCK.

Form Fields (all required unless marked optional)

FieldTypeNotes
DatedateDefaults to today. Editable.

Unloaded BytextStaff name. Auto-filled with logged-in user's name.
Supplier / Invoice ReftextOptional. Supplier name or purchase invoice number.
Serial Numbersscan + manualMultiple serials. Each scanned or typed individually. See scanner spec below.

Scanner Behaviour

  • Camera opens using device rear camera via browser/app barcode API
  • Supports standard barcodes (Code 128, Code 39, EAN-13) and QR codes
  • Each successful scan adds the serial as a chip/tag on screen with a remove button
  • Duplicate serial within the same entry triggers a warning toast — not added twice
  • Manual text input fallback: type serial and press Enter to add
  • Camera stays open between scans — staff does not need to re-open after each unit
  • Scan count shown live: 14 serial numbers added

On Save

  • Each serial number creates one row in the Inventory sheet with status IN STOCK
  • One row created in the Inward Log sheet per serial number
  • If a serial number already exists in inventory, show error and skip that serial — do not overwrite
  • Success toast shows count: ✓ 18 items saved to inventory
  • Form clears after successful save. Date resets to today.

Serial Verification Rules

  • If scanned serial is not found in inventory → error: "Serial not in inventory" — not added
  • If scanned serial is found but status is not IN STOCK → error: "Already dispatched" — not added
  • Only serials with status IN STOCK can be added to an outward entry
  • On confirm dispatch, each serial's status updates to SOLD or DISPATCHED

Action Buttons

  • Preview & Print Challan — opens challan modal with live data. Can be tapped before or after confirming dispatch.
  • Confirm Dispatch — saves dispatch, updates all serial statuses, writes to Google Sheets.
07 — DISPATCH CHALLAN

Dispatch Challan (Print)

A print-ready A4 document generated at the time of outward dispatch. Staff can preview it in a modal and then print or save as PDF via the browser/app print dialog.

Challan Contents

  • Header: Company name, address, GSTIN, phone number (from Settings)
  • Challan number and date — top right
  • Dispatched To block: Buyer name, full address, buyer GSTIN
  • Details block: Date, transport mode, handed over by, total item count
  • Items table: Sr. No. | Serial Number | Brand | Model No. | Category | HSN Code
  • Total row: Total items count at bottom of table
  • GST Summary table (if price entered): Taxable value, CGST %, SGST %, Grand Total
  • Remarks section
  • Authorised Signatory box with staff name + receiver signature box
  • Footer: Computer-generated disclaimer + company watermark

Print Behaviour

  • When print is triggered, only the challan is sent to the printer — all app UI is hidden
  • Formatted for A4 paper with 10mm margins
  • "Save as PDF" works via standard browser print dialog — no extra software needed
  • Challan is also accessible from the Outward Log in the admin panel (reprint previous challans)
✅
Light theme for print

The challan prints on a white background with black text regardless of the app theme. It must be professional and clean — no dark backgrounds, no colored fills on the main body.

11 — GOOGLE SHEETS INTEGRATION

Google Sheets Sync

Write Operations

  • Every inward save appends rows to Inventory tab and Inward Log tab via Sheets API v4
  • Every outward save updates existing rows in Inventory tab (status, buyer, date out) and appends to Outward Log
  • Every action appends to Staff Log tab
  • Writes happen asynchronously after the local save — app does not block on network
  • If write fails (no internet), entry is queued locally and retried automatically when connection resumes

Read Operations

  • App primarily reads from local storage (IndexedDB or localStorage) for speed
  • On app load, admin can trigger a full sync from Sheets to overwrite local data
  • Serial number lookup (to verify in-stock status on outward scan) reads from local data only — must be fast

Auth Method

  • Use Google Sheets API v4 with a service account JSON key (recommended for write access without user OAuth)
  • The service account email is added as an Editor to the Google Sheet
  • Alternatively: OAuth2 with the admin's Google account — admin logs in once and grants access
  • API key alone (read-only) is insufficient — write access requires service account or OAuth

Android App Strategy

Build the PWA first. Wrap it in a WebView-based Android app using Android Studio. The same codebase powers both — this avoids maintaining two separate codebases and keeps cost low. Native features needed: camera access (WebView supports this), and an APK for Play Store or direct APK install.

13 — NON-FUNCTIONAL REQUIREMENTS

Non-Functional Requirements

Performance

  • App must load in under 3 seconds on a mid-range Android phone on 4G
  • Serial number verification (in-stock check) must respond in under 200ms — reads from local data only
  • Camera scanner must start in under 2 seconds after tap
  • Handles 50,000+ inventory rows without table lag

Offline Support

  • Inward and outward forms must work fully offline — data saves locally first
  • Unsynced entries are clearly marked with a sync-pending indicator
  • Auto-sync when network is restored — no manual intervention required
  • Admin dashboard requires network to show live Google Sheets data

Design — Light Theme

  • White/light grey background throughout — no dark mode for warehouse use (bright screens easier to read in godown lighting)
  • Primary accent colour: deep blue (#1a56db) for buttons and highlights
  • Status colours: Green = IN STOCK, Red = SOLD, Amber = DISPATCHED
  • Clean, high-contrast typography — minimum 14px body text for readability on phones
  • Tap targets minimum 44×44px — staff will be using gloves or handling units while tapping

Security

  • All API keys stored server-side or in environment variables — never in client-side JS
  • Employee sessions expire after 8 hours of inactivity
  • Employee can only access data endpoints relevant to their permitted actions
  • Google Sheets service account key never exposed to the browser

Compatibility

  • Chrome on Android (primary) — must be fully functional
  • Safari on iOS — must work for inward/outward scanning
  • Desktop Chrome — admin panel fully functional
  • Android APK targets API level 26+ (Android 8.0 and above)
